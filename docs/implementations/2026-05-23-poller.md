## тЭп Problem / Motivation

Build step 8 of 20 тАФ the last piece of Phase 1 (ingestion). Steps 4-7
delivered four per-kind ingesters (RSS, HTML, Resm├о Gazete PDF, Reddit
PRAW) that each conform to the `Ingester` Protocol defined in step 4.
Step 8 binds them into one nightly orchestration:

- One uniform fan-out across all 37 sources in the registry.
- One concurrency cap so the local box does not open dozens of TCP
  connections at once.
- One per-source timeout so a hung response never stalls the night.
- One `ingest_log` row per source so the morning briefing's S─░STEM LOG
  footer (ADR-009) has the source-by-source health record.
- One `pipeline_runs` row so the rest of the pipeline (normalize,
  cluster, score, arc-link, write) can advance ``stages_done`` and
  eventually mark the run ``COMPLETED``.

Per ADR-012 the principle is *degrade gracefully, never abort*: a single
source's timeout or exception must never collapse the run. The
Protocol's contract is "fetch returns IngestResult, doesn't raise for
expected failures" тАФ the poller is the outer safety net for the
unexpected failures (DB errors, surprise traceback in a future ingester
edit, etc.).

## тЭп Implementation Approach

### Dispatch тАФ `get_ingester(source, conn, settings=None)`

```python
if   source.kind is SourceKind.RSS:  return RssIngester(conn=conn)
elif source.kind is SourceKind.HTML: return HtmlIngester(conn=conn)
elif source.kind is SourceKind.PDF:  return ResmiGazeteIngester(conn=conn)
elif source.kind is SourceKind.API:  return RedditIngester(conn=conn, settings=settings)
else:                                return None  # logged warning
```

`SourceKind.PDF` currently means Resm├о Gazete only. The dispatcher's
inline comment flags this тАФ adding a second PDF source means branching
by `source.id` here, not silently aliasing every PDF to the Gazette
implementation.

`DEFERRED` (and any future unknown kind) returns ``None`` with a logged
warning. The poller then writes a `SKIPPED` ingest_log row тАФ that's the
designed "we know this kind exists but cannot ingest it yet" path.

### Adding a new SourceKind

The Ingester Protocol is the contract; conformance requires only an
``async def fetch(source) -> IngestResult`` method that never raises for
expected failures. Steps:

1. Add the enum member to `musahit/common/types.SourceKind` тАФ
   **this is an ADR amendment**, per the project convention (`MEMORY.md`).
2. Create the ingester module under `musahit/ingest/` and a test file.
3. Branch on the new enum value in `get_ingester`. (This file is
   FILE-PROTECTED; the branch addition needs the operator override too.)
4. Update `sources.py` to add the registry rows. (FILE-PROTECTED; same
   override discipline.)

Each `fetch_one_source`:

1. Acquires a semaphore slot.
2. Looks up the ingester via the factory (production: `get_ingester`;
   tests: an injected callable).
3. If no ingester: writes SKIPPED ingest_log row and returns.
4. Records `started_at = utcnow()` (naive UTC).
5. Calls `await asyncio.wait_for(ingester.fetch(source), timeout=тАж)`.
6. Catches `TimeoutError` тЖТ `IngestResult(status=TIMEOUT, тАж)`. Catches
   any other `Exception` тЖТ `IngestResult(status=PARSE_ERROR, тАж)`.
7. Records `completed_at = utcnow()`.
8. Writes the ingest_log row (UPSERT).
9. Releases the semaphore slot.

### Per-source timeout floor

```python
timeout = max(default_timeout_seconds, source.rate_limit_seconds * 12)
```

A source with a 10s rate limit needs more than 10s to make a single
fetch, so the poller's cap must be higher than `rate_limit_seconds`.
The 12├Ч multiplier is generous enough that an RSS feed with a 30s
rate limit gets a 360s ceiling тАФ enough for slow downloads but not
unbounded. The ``default_timeout_seconds=60`` floor handles fast-
rate-limit sources.

### `pipeline_runs` and `ingest_log` row lifecycle

| Column / Field | Owner stage | When written | Final value |
|---|---|---|---|
| `pipeline_runs.run_id` | poller | `run()` start | run_id arg or default |
| `pipeline_runs.started_at` | poller | `run()` start | naive UTC `utcnow()` |
| `pipeline_runs.status` | poller (writes RUNNING); writer (final COMPLETED) | poller upsert | RUNNING here; COMPLETED later |
| `pipeline_runs.stages_done` | each stage appends | poller writes `[]` at start, `["ingest"]` at end | grows per stage |
| `pipeline_runs.counts` | each stage adds keys | poller writes `{"articles": N}` | normalize adds `articles_normalized`, etc. |
| `pipeline_runs.completed_at` | writer stage | final stage | naive UTC |
| `ingest_log.run_id` | poller | per source | same as pipeline_runs.run_id |
| `ingest_log.source_id` | poller | per source | source.id |
| `ingest_log.started_at` | poller | per source | naive UTC at slot acquisition |
| `ingest_log.completed_at` | poller | per source | naive UTC after fetch (or timeout/exception) |
| `ingest_log.status` | poller | per source | mapped from IngestResult.status |
| `ingest_log.articles_fetched` | poller | per source | IngestResult.count |
| `ingest_log.error_detail` | poller | per source | IngestResult.error or None |

The poller writes `ingest_log` rows via `ON CONFLICT (run_id, source_id)
DO UPDATE` тАФ a rerun of the same run_id overwrites the existing rows
rather than appending duplicates.

The `pipeline_runs` row is similarly upserted: re-running the same
`run_id` resets `started_at`, `completed_at`, `stages_done`, and
`counts`. The row represents the *current* attempt, not a historical
aggregate.

### Concurrency cap rationale (default = 8)

Eight simultaneous fetches:

- Pulls within the operator's residential bandwidth without saturating.
- Stays well under typical OS file-descriptor budgets even with the per-
  fetch HTTPS connection plus the Reddit thread's overhead.
- Is large enough that a single hung source does not block the whole
  fan-out тАФ even at the per-source timeout ceiling of 60s+, the
  remaining seven slots keep processing.

Configurable via the constructor: tests use `max_concurrent=3` to verify
the cap is actually enforced (the `ConcurrencyTracker` test fixture
asserts `peak тЙд 3` and `peak тЙе 2`).

### Failure isolation summary

| Failure class | Where caught | Resulting `IngestResult` | Poller action |
|---|---|---|---|
| HTTP error inside ingester | the ingester | `HTTP_ERROR` | log + write row + continue |
| Timeout inside ingester | the ingester | `TIMEOUT` | log + write row + continue |
| Parse error inside ingester | the ingester | `PARSE_ERROR` | log + write row + continue |
| Per-source `asyncio.wait_for` timeout | poller | `TIMEOUT` (poller-synthesized) | log + write row + continue |
| Any other exception in `fetch` | poller | `PARSE_ERROR` (poller-synthesized) | log + write row + continue |
| Reddit credentials missing | the ingester | `SKIPPED` | log + write row + continue |
| Unknown SourceKind | the poller (no ingester) | `SKIPPED` | log + write row + continue |

## тЭп Design Decisions

**`asyncio.gather(return_exceptions=False)`.**
The per-source coroutine `_run_one_source` already wraps everything in
try/except and never raises. With that invariant, `gather` cannot
surface an exception тАФ passing `return_exceptions=True` would just hide
a programming error.

**Per-source UPSERT for `ingest_log`.**
Rerunning the same `run_id` should update the row, not insert a
duplicate. The PRIMARY KEY is `(run_id, source_id)`; `ON CONFLICT
тАж DO UPDATE` is the natural fit.

**Status stays `RUNNING` after ingest finishes.**
The full pipeline is a chain of stages (ingest тЖТ normalize тЖТ cluster тЖТ
score тЖТ arc-link тЖТ write). Marking `COMPLETED` after the first stage
would lie about the run's state. The writer stage flips
`COMPLETED`/`FAILED` at the very end. (`ADR-007 ┬з Liveness` depends on
this discipline тАФ the 06:45 liveness probe checks
`pipeline_runs.status = COMPLETED`.)

**`get_ingester` returns plain `Ingester | None`, not a richer type.**
A None return path is simpler than ``raise NotImplementedError`` and
matches the goal's wording. The caller (`_fetch_one`) translates None
into a SKIPPED ingest_log row.

**Concurrency cap = 8 (default).**
Empirical тАФ small enough that a residential pipe handles all eight
fetches cleanly, large enough that a hung source doesn't drain the
slot pool. The test uses 3 to keep the timing window small.

**Per-source rate_limit_seconds ├Ч 12 floor.**
12├Ч is enough room for a feed with a long rate limit to actually
download. ADR-003 mentions rate limits as politeness, not as proof of
download time, but the 12├Ч floor captures both: a slow feed gets
extra room without making the timeout unbounded.

**FILE-PROTECTED marker at the top of the file.**
This module joins `sources.py`, `defcon.py`, and `promotion.py` on the
protected list. Future edits require an ADR amendment and explicit
operator override тАФ same discipline as other registry/decision files.

## тЭп Operator caveats

- **A misbehaving ingester still consumes a slot until timeout.**
  The poller waits up to `max(60s, rate_limit * 12)` per source. Eight
  simultaneously misbehaving sources would idle the pipeline for that
  full window. In practice this is fine тАФ the nightly budget is
  measured in minutes, not seconds тАФ but revisit if a future source's
  rate_limit is set very high.
- **`status=RUNNING` after the poller finishes is intentional.**
  Do not panic-read a `pipeline_runs` row whose status is `RUNNING` if
  `stages_done=["ingest"]` тАФ that just means the next stage hasn't
  picked it up yet. The 06:45 liveness probe (ADR-007) does the right
  check (`status = COMPLETED`); the dashboard's run view shows the
  stage breakdown.
- **`run_id` collisions are an UPSERT.** Running the poller twice with
  the same `run_id` is intentional behavior (manual rerun); the second
  pass overwrites the first's `ingest_log` rows. To preserve history,
  pass a different `run_id` on the rerun (e.g.,
  `run_20260523_rerun_01`).
- **No per-source backoff or retry.** Step 8's contract is "best effort,
  one attempt per source per run." If transient retries become the
  right behavior, that's an ADR amendment + a wrapper around
  `ingester.fetch` inside the per-source coroutine.
- **Reddit's `asyncio.to_thread` interacts with the semaphore.** Each
  Reddit fetch holds a slot for the duration of its PRAW work (which
  can be 10+ seconds with lazy comment loads). One subreddit takes one
  slot; the four Reddit subreddits run sequentially inside
  `RedditIngester.fetch`, so the poller treats the whole Reddit run as
  a single slot.
